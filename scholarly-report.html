Abstract

The WeD project proposes the creation of a sophisticated multi-device (micro-)service-oriented cloud-based system designed to intelligently model and manage public technical content related to Web development. This encompasses a diverse range of information, including tutorials, source-code examples, news, events, and more, sourced from reputable platforms such as MDN Web Docs. The core innovation lies in the application's utilization of Linked Data principles, enabling it to generate and expose knowledge of significant interest to Web developers. This knowledge is accessible to both human users and software through a SPARQL endpoint.

The system facilitates granular exploration based on various criteria, including programming language/paradigm, specific framework guides/references, target platform, purpose. Furthermore, the project integrates additional knowledge from DBpedia and/or Wikidata, enhancing the richness and diversity of information available. The implementation aligns with the ACM Computing Classification System, ensuring comprehensive coverage of topics relevant to the Web development community.

Introduction

In the ever-evolving landscape of Web development, the abundance of technical content across different platforms necessitates a more intelligent and efficient approach to modeling and managing this wealth of information. The WeD project addresses this challenge by proposing the development of a multi-device (micro-)service-oriented system that embraces Linked Data principles.

The system's primary goal is to provide Web developers with a smart and dynamic interface for exploring and discovering public technical content. This content spans various types, from tutorials to source-code examples, news, and events, sourced from leading platforms like MDN Web Docs.

By incorporating Linked Data principles, WeD enables the generation and exposure of knowledge relevant to Web developers. The system employs a SPARQL endpoint, ensuring accessibility for both human users and software applications. Through a thoughtful implementation, users can navigate this knowledge repository based on specific criteria such as programming language, framework, target platform, purpose, geographical area, period of time, and personalized preferences.

Product Scope

The scope of the WeD (Web Developer Companion) project is defined by its overarching goal: to create a multi-device (micro-)service-oriented cloud-based system that intelligently models and manages public technical content related to web development. The system aims to cater to the diverse needs of web developers, providing a centralized and smart interface for accessing a wide range of information.

Key Features and Functionalities:

  • Aggregation of Technical Content: Collecting tutorials, source-code examples, news, events, and more from various reputable sources including MDN Web Docs.
  • Linked Data Principles: Implementation of Linked Data principles to generate and expose knowledge relevant to web developers.
  • SPARQL Endpoint: Providing a SPARQL endpoint for both human users and software applications to query and retrieve specific information.
  • Granular Exploration: Enabling users to explore content based on criteria such as programming language, framework, target platform, purpose.
  • Integration with External Knowledge Sources: Enhancing the information ecosystem by incorporating additional knowledge such as DBpedia or Wikidata.

Exclusions

The project scope explicitly excludes the development of features unrelated to web development, content from non-reputable sources, and functionalities that deviate from the overarching goal of providing a valuable resource for web developers.

Overall Description

Product Perspective

The web application WeD is a new self-contained product independent from other larger systems.

Product Functions

1. Aggregation of Technical Content

The WeD system will aggregate diverse technical content related to web development, including tutorials, source-code examples, news, events, and more. This content will be sourced from reputable platforms such as MDN Web Docs, and other relevant sources.

2. Linked Data Principles

Implementation of Linked Data principles to intelligently model and expose knowledge of interest to web developers. This approach ensures the interconnectedness and discoverability of information within the system.

3. SPARQL Endpoint

Provision of a SPARQL endpoint to enable users, both human and software applications, to query and retrieve specific information from the knowledge base. This feature enhances the accessibility and flexibility of interacting with the stored data.

4. Granular Exploration

Enable users to explore the aggregated content based on specific criteria, such as programming language, framework, target platform, purpose, geographical area, period of time, and user preferences. This granular exploration ensures users can tailor their search to meet specific needs.

5. Integration with External Knowledge Sources

Enhance the information ecosystem by integrating additional knowledge from external sources such as DBpedia and/or Wikidata. This integration contributes to the depth and diversity of information available to users.

The design and implementation of the WeD (Web Developer Companion) system are guided by a set of considerations to ensure optimal performance and functionality. One primary constraint is the need for careful data handling and processing, given the large volume of technical content sourced from diverse platforms. While the WeD system has no strict restrictions regarding the technologies used, consideration will be given to industry best practices and compatibility with widely adopted web standards. Internal Data Structures and Models

Designing a robust internal data structure is fundamental to the success of the WeD (Web Developer Companion) project. This section outlines the preliminary considerations for the data structures and models that will underpin the system's architecture, ensuring efficient storage, retrieval, and manipulation of web development content.

The internal data model will be based on the Resource Description Framework (RDF) to represent information in a semantic and interconnected manner. RDF provides a flexible and extensible framework for defining entities, relationships, and metadata within the knowledge base.

Consideration will be given to defining explicit relationships between entities, ensuring a rich network of connections. This approach enables users to traverse the knowledge base seamlessly and discover related content based on various criteria such as topic, language, and framework.

The incorporation of ontologies will play a crucial role in structuring concepts within the data model. Ontologies provide a shared vocabulary that enhances the clarity and consistency of information, promoting a standardized understanding of web development content.

Each entity and resource will be assigned a Unique Resource Identifier (URI) to ensure globally unique identification. URIs enable unambiguous referencing and facilitate interoperability with external datasets, aligning with Linked Data principles.

Conformance to Linked Data Principles

The WeD (Web Developer Companion) project meticulously adheres to the Linked Data principles, fostering a structured and interconnected knowledge representation. These principles guide the design and implementation of the system, ensuring seamless integration, discoverability, and meaningful relationships within the web of data.

Each entity and resource within the WeD system is assigned a unique Uniform Resource Identifier (URI). This ensures unambiguous identification and facilitates global accessibility to resources.

Resource Description Framework (RDF) is employed as the foundational language for representing knowledge in a machine-readable format. This allows for the creation of semantic relationships and enriches the content with meaningful context.

The Hypertext Transfer Protocol (HTTP) is utilized for data retrieval and interaction within the WeD system. This standardized protocol ensures uniform access to resources and supports the principles of openness and decentralization.

WeD leverages RDF as the standard data model, promoting the creation of links between resources. The SPARQL query language is employed to provide users and applications with a flexible and powerful means of querying and extracting relevant information.

The WeD system integrates additional knowledge from external sources such as DBpedia and/or Wikidata. This cross-referencing enhances the richness of information, aligning with the Linked Data principle of connecting to other datasets.

The project incorporates ontologies to define and organize concepts within the knowledge base. This semantic structuring ensures a shared understanding of data and facilitates more nuanced and accurate queries.

Technical Aspects

As previously mentioned, our system leverages a microservice architecture, utilizing various Azure services and programming languages tailored to each sub-task. The choice of Azure for cloud hosting and the integration of CI/CD processes using Bicep and Azure Pipelines are central to the project's infrastructure and operational efficiency

Microservice Architecture and Azure Services

The use of a microservice architecture is significant in managing the system's complexity, ensuring scalability, and facilitating the development process by breaking down tasks into manageable, independent services. Each microservice performs a specific role within the system, enhancing maintainability and allowing for the use of different programming languages and technologies that are best suited for each task.

Importer

This Azure Function App is responsible for scraping and parsing content from sources like MDN into JSON format. It leverages a Redis Cache to efficiently manage the high volume of data processed. The parsed data is then published to an Azure Service Bus queue, allowing for asynchronous data processing and decoupling from subsequent services

Enricher

Also implemented as an Azure Function App, the Enricher receives JSON data from the Azure Service Bus, enriches this data into JSON-LD format—a key step for integrating Linked Data principles—and forwards it to another Service Bus queue. This enrichment process is crucial for semantic web compatibility and enhances the data's value by adding context and meaning.

Converter

The final step in the data processing pipeline, this Azure Function App converts the JSON-LD into RDF data using the Apache Jena library. The RDF data is then stored in a Fuseki RDF Store hosted on an Azure Virtual Machine, making the data ready for querying using SPARQL.

API and Web Application

The system's API, developed with Java Spring Boot and hosted on an Azure App Service, interfaces with the Fuseki RDF Store to query data using SPARQL endpoints. It also exposes its endpoints to the web application, facilitating data access and manipulation. The web application, built with React and hosted on Azure Static Web Apps, provides an intuitive platform for users to interact with the system, accessing and visualizing data according to their preferences.

CI/CD Infrastructure

Using Azure DevOps and a combination of Bicep templates and Azure YAML pipelines, the project achieves automated deployments, testing, and infrastructure management. This CI/CD setup ensures that updates, features, and fixes are rapidly and reliably deployed across all components of the system, from microservices to the web application, while also facilitating rollback and version control. The extensive use of Infrastructure as Code (IaC) with Bicep allows for reproducible and consistent infrastructure provisioning, reducing the potential for human error and enhancing security and compliance.
